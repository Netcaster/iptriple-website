How It Works

Three Steps from Document to Answer

01 — INGEST
📥

Ingest

Client documents, web pages, PDFs, and internal wikis are chunked into text segments and converted into vector embeddings. Firecrawl handles web crawling — up to 10,000 pages per knowledge base — with support for PDF, DOCX, XLSX, TXT, CSV, and S3-compatible storage.

→
02 — STORE
🗄️

Store

Embeddings are stored in Qdrant — a self-hosted vector database running in Docker on IP Triple's DigitalOcean infrastructure. Each client gets a fully isolated collection with hybrid lexical and semantic retrieval. No cross-tenant data leakage, ever.

→
03 — GENERATE
⚡

Retrieve & Generate

When a user submits a query via the NaluAsk embed tag, the system retrieves the most relevant chunks from the client's vector database, injects them into the prompt context, and routes to Claude, ChatGPT, or Perplexity — producing accurate, brand-aware, source-grounded answers in ~200ms.
